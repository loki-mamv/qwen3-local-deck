02 — Overview

What Is Qwen3 Coder Next?

  • 80B total parameters, but only 3B active per token
  • Ultra-sparse Mixture-of-Experts (MoE) architecture
  • At Q4 quantization: ~46GB model size
  • 256K context window (131K practical for most setups)
  • Benchmarks rival models with 10-20x more active params
  • Open weights — run it yourself, no API costs
03 — Motivation

Why Run Locally?

  • No API costs — $0/month vs $100+/month for Claude Max
  • Full privacy — code never leaves your machine
  • No rate limits or throttling
  • Run multiple parallel agents (coding swarm)
  • One-time hardware cost pays for itself in 3-6 months
04 — Understanding Performance

The Bottleneck —
Memory Bandwidth

  • MoE models load different expert weights per token
  • GPU compute is NOT the bottleneck — memory bandwidth is
  • More VRAM = more experts cached = faster inference
  • System RAM matters too — experts offload to DDR5
  • NVMe speed matters for cold expert loads

07 — Software

Software Stack

  • llama-server (llama.cpp) for inference
  • MXFP4 or Q4_K_M quantization
llama-server -ngl 999 -fa on -cmoe -c 131072
  • Point Claude Code or OpenCode at localhost:8080
  • vLLM for higher throughput (requires more VRAM)
  • Docker: nvcr.io/nvidia/vllm container
